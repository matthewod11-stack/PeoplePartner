## Security Details

### S1: SQL Injection in Employee Listing Filter (HIGH)

**Location:** `src-tauri/src/employees.rs:317-341`

The `list_employees` function builds SQL WHERE clauses using string formatting with user-supplied filter values. While single quotes are escaped with `replace`, this is fragile and inconsistent with parameterized queries used everywhere else.

**Remediation:** Refactor to use parameterized queries with bind variables, consistent with the rest of the codebase.

### S2: API Key Stored as Plaintext File (HIGH)

**Location:** `src-tauri/src/keyring.rs:30-73`

Despite the UI claiming "Stored securely in macOS Keychain" (`ApiKeyInput.tsx:112`), the API key is stored as a plaintext file at `~/Library/Application Support/com.hrcommandcenter.app/.api_key` with 0o600 permissions. A TODO comment on line 4 acknowledges this.

**Remediation:** Use the `security-framework` crate for actual macOS Keychain integration. Update the UI text until migration is complete. Remove the `println!` on line 72 that logs the key path.

### S3: Content Security Policy Disabled (HIGH)

**Location:** `src-tauri/tauri.conf.json:27`

CSP is set to `null`, disabling all Content Security Policy protections. Combined with Markdown rendering of AI responses (S4), this creates an XSS attack surface.

**Remediation:** Configure a restrictive CSP:
```json
"security": {
    "csp": "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; connect-src 'self' https://api.anthropic.com; img-src 'self' data:"
}
```

### S4: XSS via Markdown Rendering (MEDIUM)

**Location:** `src/components/chat/MessageBubble.tsx:109-111`

`ReactMarkdown` with `remarkGfm` but no `rehype-sanitize`. With CSP disabled, edge-case Markdown constructs could be exploited.

**Remediation:** Add `rehype-sanitize` as an explicit sanitization layer to the rehypePlugins array.

### S5: Database Not Encrypted at Rest (MEDIUM)

**Location:** `src-tauri/src/db.rs:39`

All employee PII stored unencrypted. CLAUDE.md acknowledges this as deliberate ("OS sandbox only"). Relevant for GDPR/CCPA compliance.

### S6: PII Scanner Limited to Financial Data (MEDIUM)

**Location:** `src-tauri/src/pii.rs:1-10`

Only detects SSN, credit cards, bank accounts. Misses phone numbers, addresses, medical info.

### S7: Audit Log Stores Full Response Text (MEDIUM)

**Location:** `src-tauri/src/audit.rs:57-66`

Full AI response text (which may contain employee details) stored and exportable via CSV.

### S8: Raw Error Messages Forwarded (LOW)

**Location:** `src-tauri/src/employees.rs:28-35`

`EmployeeError::Database` forwards raw SQLx error strings to frontend.

### Positive Security Observations

- Parameterized queries used correctly everywhere except `list_employees` filter
- Encrypted backups use AES-256-GCM + Argon2id
- No unsafe innerHTML usage anywhere in frontend
- PII redaction before API calls with audit logging
- User messages rendered as plain text (not Markdown)
- FTS query sanitization in `prepare_fts_query`

## Performance Details

### P1: Streaming Causes Full-Tree Re-renders (CRITICAL)

**Location:** `src/contexts/ConversationContext.tsx:396-406`

During streaming, every chunk triggers `setMessages(prev => prev.map(...))`, creating a new array reference. Since `ConversationContext` provides a single flat object, every consumer re-renders at 20-50 Hz — including unrelated components like `ConversationSidebar`.

**Remediation:** Split into `ConversationDataContext` (messages) and `ConversationActionsContext` (stable functions). Throttle `setMessages` to ~10 updates/sec via `requestAnimationFrame`.

### P2: N+1 Rating Queries (HIGH)

**Location:** `src/contexts/EmployeeContext.tsx:140-150`

After fetching employees, code fires `Promise.all(result.employees.map(emp => getLatestRating(emp.id)))` — up to 200 individual IPC calls.

**Remediation:** Create single `list_employees_with_ratings` Rust command with SQL JOIN.

### P3: No Code Splitting (HIGH)

**Location:** `vite.config.ts`

Entire app compiles to single 1.24 MB JS bundle. Import-heavy components like `ImportWizard`, `InsightBoardView`, `CommandPalette` loaded upfront.

**Remediation:** `React.lazy()` + `Suspense` for infrequent views. Configure `manualChunks` for `recharts`, `react-markdown`, `fuse.js`.

### P4: Context Value Recreated Every Render (HIGH)

**Location:** Both context providers

New object literal on every render means all consumers re-render even when no data changed.

**Remediation:** Wrap each context value in `useMemo`.

## Accessibility Details

### A1: Modals Lack Focus Trap + ARIA (CRITICAL)

**WCAG:** 2.4.3, 4.1.2 (Level A)
**Location:** `ImportWizard.tsx:212-311`, `EmployeeEdit.tsx:219-396`, `ConversationSidebar.tsx:196-236`

Built as raw `<div>` overlays without `role="dialog"`, `aria-modal`, or focus trap. Tab key escapes to hidden content behind modal.

**Remediation:** Use the existing shared `<Modal>` component (which has proper ARIA) or add `focus-trap-react`.

### A2: Charts Have No Screen Reader Alternative (CRITICAL)

**WCAG:** 1.1.1, 1.3.1 (Level A)
**Location:** `AnalyticsChart.tsx:134-191`

Recharts SVGs render with no textual alternative. The entire analytics feature is invisible to screen readers.

**Remediation:** Add `role="img"` + `aria-label` summarizing chart data. Provide visually-hidden data table fallback.

### A3: Drilldown Rows Not Keyboard-Accessible (CRITICAL)

**WCAG:** 4.1.2, 2.1.1 (Level A)
**Location:** `DrilldownModal.tsx:101-126`

`<tr>` elements have `onClick` but no `tabIndex`, `role="button"`, or `onKeyDown`.

### A4: Shared Modal Missing Focus Trap (IMPORTANT)

**WCAG:** 2.4.3 (Level A)
**Location:** `Modal.tsx:56-141`

Has correct `role="dialog"` and `aria-modal` but Tab can still escape.

### A5: Color Contrast Failures (IMPORTANT)

**WCAG:** 1.4.3 (Level AA)

`stone-400` on `stone-50` = ~2.6:1 (fails). `stone-500` on `stone-50` = ~4.0:1 (fails for normal text). Note: V2.2.5a previously addressed some contrast issues but remaining gaps identified.

### A6-A10: See summary table above.

### Positive Accessibility Observations

- Shared Modal has `role="dialog"`, `aria-modal`, `aria-labelledby`
- ChatInput has `aria-label` on textarea and send button
- CommandPalette implements combobox pattern with `aria-expanded`
- FileDropzone has keyboard handling, `role="button"`, `tabIndex`
- Avatar uses `aria-label={name}`
- Settings toggles use `role="switch"` and `aria-checked`
